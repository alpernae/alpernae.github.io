Textpattern 4.8.8 Remote Code Execution (RCE) via Unrestricted File Upload Vulnerability

Hello hacker friends, Today I am writing about the vulnerability I found on textpattern. Textpattern is a content management system and open source. I had previously found an XSS vulnerability on a version of the textpattern.

Today, I did a vulnerability search on the most current version (4.8.8) and found the Remote Code Execution vulnerability, let's move on to the technical details. First, let's run our Burp Suite tool and open our proxy browser located inside the burp. I had set up the CMS at my own localhost address for testing, so I go directly to localhost and then start monitoring all requests. After logging in to the admin panel, I explore the file upload page, I try to find out which directory the file I uploaded was uploaded to by uploading a sample file on it. When I saw that the files were uploaded as /files/sample.file, I tried to access the file I uploaded by logging out of the admin account without logging in, as a result, I saw that I could access it, after this step, I tried to log back into the admin account and upload the single line backdoor code written in php language and I saw that it was uploaded.



As you can see in the picture above, the attacker has uploaded the code with the content of such PHP files need to be uploaded through a more reliable way because as a result of the hijacking of an admin account, the attacker can take over the server to upload the attack codes. Now let's move on to the second stage as an attacker, in the examinations I made before, we could access the file we uploaded from the outside, so we can access the backdoor we installed from the outside, so we can open a browser and type adress line target.com/files/attackfile?cmd={OS_COMMANDS} as an attacker I can run any command on the server.
